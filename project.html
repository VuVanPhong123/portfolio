Project 1: No mercy!!!

Project Overview

A fast-paced 2D arcade game developed using SDL (Simple DirectMedia Layer) library in C++. Inspired by "NO HUMANITY", this game challenges players to control a UFO and dodge incoming obstacles across 6 increasingly difficult levels.

Key Features:

  • 6 Unique Levels: Each with different obstacle patterns and mechanics
  • Precise Controls: Arrow key navigation with collision detection
  • Dynamic Obstacles: Meteors, clouds, flying fans, tornadoes, and animal enemies
  • State Management: Single variable (int a) controls all game states
  • Retry System: Option to restart levels after failure

Technical Implementation:

Core Game Loop Structure:


                                int a = 0; // Main state variable

                                while (gameRunning) {
                                    if (a == 0) { /* Waiting screen */ }
                                    else if (a == 1) { /* Level 1 */ }
                                    // ... up to a == 18 for all states
                                }

Collision Detection:


                                bool Detecting_Collision(SDL_Rect& object, SDL_Rect& UFO) {
                                    if (object.y+object.h < UFO.y || 
                                        object.y > UFO.y+UFO.h || 
                                        object.x > UFO.x+UFO.w || 
                                        object.x+object.w < UFO.x)
                                        return false;
                                    return true;
                                }

Level Breakdown:

Level 1 - Meteor Shower

  • Obstacles: 9 randomly spawning meteors
  • Behavior: Appear from screen edges and move toward UFO
  • Challenge: Basic pattern recognition and quick reflexes
  • Code: Uses MeteoroidRect array and SpawningMeteoroid() function

Level 2 - Cloud Fall

  • Obstacles: 10 falling clouds with 1 random gap
  • Behavior: Spawn at top, fall vertically with random disappearance
  • Challenge: Identify safe path through cloud formation
  • Code: Uses CloudRect array and RandomMissingCloud variable

Level 3 - Flying Fans

  • Obstacles: 4 animated flying objects
  • Behavior: Horizontal movement with 4-frame animation
  • Special: Spawn homing missiles when aligned with UFO
  • Explosion: Creates 6 fragments upon collision detection
  • Code: Uses movingFlyingFan() and movingFragment() functions

Level 4 - Tornado Alley

  • Obstacles: 5 expanding tornadoes
  • Behavior: Spawn at corners, expand to maximum size
  • Challenge: Navigate through dynamic obstacle sizes
  • Code: Reuses meteoroid mechanics with different visuals

Level 5 - Animal Attack

  • Obstacles: Cats and dogs combination
  • Behavior:
    • Dogs: Meteor-like movement patterns
    • Cats: Cloud-like falling behavior
  • Challenge: Dual-pattern recognition and avoidance
  • Code: Uses spawningDogCat() and movingCat() functions

Final Level - Ultimate Challenge

  • Obstacles: All previous enemies combined
  • Behavior: Meteor showers + cloud gaps + homing missiles
  • Challenge: Comprehensive skill test with no mercy
  • Special: 2x speed multiplier for intense gameplay

Development Notes:

Technical Challenges

  • Single variable state management (int a) for all game screens
  • Optimized collision detection with rectangular hitboxes
  • Memory management for multiple spawned objects

Design Choices

  • Progressive difficulty curve across levels
  • Reusing mechanics with visual variations
  • Minimalist UI to maintain focus on gameplay

Project 2: Foody - Street Vendor Order App

Project Overview

Foody is a lightweight mobile-friendly web app built with Kivy, designed for street vendors to manage orders and for customers to place orders remotely. This project helps small food vendors streamline order tracking, inventory management, and customer interaction easily.

Key Features:

  • Dual App Design: Owner app for order and inventory management, customer app for browsing and ordering.
  • Order Tracking: View orders, customer addresses, and daily income easily.
  • Ingredient Management: Track ingredients, expiration, and costs for effective management.
  • Rating System: Customers can rate food items post-order.

Technical Implementation:

Sample Order Tracking Logic:

orders = []
                    def add_order(customer, items):
                        order = {"customer": customer, "items": items}
                        orders.append(order)

Ingredient Tracking Example:

ingredients = {"bread": 20, "pork": 15}
                    def update_stock(item, qty):
                        if item in ingredients:
                            ingredients[item] -= qty

Development Notes:

  • Used Kivy for cross-platform GUI development.
  • Focus on low-memory usage to run on low-cost devices.
  • Simple and clean UI/UX for street vendor needs.

Project 3: Bank Customer Churn Prediction

Project Overview

This project implements a Logistic Regression classifier from scratch to predict bank customer churn. Using advanced techniques like momentum-based gradient descent, regularization, and early stopping, the model identifies customers likely to leave, helping banks improve customer retention.

Key Features:

  • From-Scratch Implementation: Built Logistic Regression with numpy, manual weight updates, and sigmoid activation.
  • Momentum and Early Stopping: Includes momentum to accelerate convergence and patience-based early stopping for efficiency.
  • Regularization: Uses L2 regularization to prevent overfitting.
  • EDA: Used Seaborn and Matplotlib to analyze customer patterns.

Technical Implementation:

Logistic Regression Core Logic:


                        def sigmoid(z):
                            return 1 / (1 + np.exp(-z))

                        def compute_loss(y, y_pred, weights, reg_lambda):
                            m = len(y)
                            loss = (-1/m) * np.sum(y*np.log(y_pred) + (1-y)*np.log(1 - y_pred))
                            reg = (reg_lambda / (2*m)) * np.sum(weights**2)
                            return loss + reg

Momentum Update Example:


                        velocity_w = momentum * velocity_w - learning_rate * dw
                        weights += velocity_w

  • Trained the model on a Kaggle bank churn dataset with engineered features.
  • Tested and monitored loss to avoid overfitting.
  • Visualized churn correlations using heatmaps and pairplots.
